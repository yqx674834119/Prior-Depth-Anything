# 2026-03-23 仓库整理记录

## 1. `.pyc` / `__pycache__` 是否应该推送到 GitHub

不建议。

原因：

- 这类文件是 Python 解释器生成的缓存，不是源码
- 它们和本地 Python 版本、平台、运行时状态强相关
- 推到仓库会制造无意义 diff，也容易污染主线

本次处理：

- 已在 `.gitignore` 中保留并收敛 Python 缓存忽略规则
- 已将通用 `*.egg-info/` 加入忽略
- 后续应将已被 git 跟踪的 `.pyc` / `__pycache__` 从版本控制中移除

## 2. `prior_depth_anything.egg-info` 是什么

这是 `setuptools` 在本地安装或打包时生成的元数据目录，不是核心源码。

常见内容包括：

- 包名
- 版本
- 依赖列表
- 入口脚本
- 文件清单

结论：

- 它通常不应该提交到 GitHub
- 当前仓库已经忽略这类目录，后续保持不提交即可

## 3. `vggt` 目前有没有用到

有用到，但不是 Android / ESP32 这条主线的必要部分。

当前情况：

- `enhance_depth.py` 会直接导入 `vggt`
- `README.md` 也保留了 `VGGT` 相关说明
- `prior_depth_anything/depth_completion.py` 里有 `VGGT` 替代实现示例，但当前默认执行路径仍然是 `Depth Anything V2`

结论：

- 对 Android + ESP32 + 本地 ONNX 推理主线来说，`vggt` 不是必需
- 对 Python 侧增强/实验脚本来说，`vggt` 仍然是有效依赖

## 4. 当前整理原则

- Android / ESP32 主线优先保留
- Python 缓存与打包元数据不入库
