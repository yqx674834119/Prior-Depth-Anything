# 2026-03-23 安卓与 ESP32 通信接口冻结

## 背景

- 目前已经确认 `resource/DataFusion/demo/batch_output_v2/1647156045.486036` 这条离线验证链路可运行。
- Android 端当前主流程已经切换为“ESP32 提供图像 + ToF 深度 + 手机本地 ONNX 推理”。
- 为避免后续再次混入旧的 UDP 试验接口，本次将安卓侧的设备接口固定为一套明确规范。

## 固定后的接口定义

### 1. Wi-Fi 接入

- 热点名称：`ToFSense_AP`
- 默认主机地址：`192.168.4.1`

### 2. 相机图像接口

- 协议：HTTP
- 地址：`http://192.168.4.1:81/`
- 方法：`GET`
- 返回：单帧 JPEG 图片
- 安卓行为：扫描页以轮询方式拉取图像，当前轮询间隔约 `33ms`

### 3. ToF 深度接口

- 协议：TCP
- 地址：`192.168.4.1:8080`
- 编码：UTF-8 文本
- 数据帧：单行 JSON，以换行符结束

示例：

```json
{"d":[1234,1228,1219, ... 共64个值]}
```

约束：

- 键名固定为 `d`
- 数组长度固定为 `64`
- 数据顺序固定为 `8 x 8` 行优先
- 单位固定为 `mm`
- 安卓侧当前会过滤 `<= 0` 或 `> 9999` 的值

### 4. 安卓推理链路

- 扫描页从 ESP32 获取：
  - 图像：HTTP 81 端口
  - 深度：TCP 8080 端口
- 点击拍摄后，将当前 RGB 图像与 64 点 ToF 深度写入 `SharedData`
- 处理页使用 `prior_depth_anything_vits_fp16.onnx` 做本地推理
- 推理完成后输出 `last_rgb.png` 与 `last_depth.png`，再交给点云页展示

- 首页旧的“点连接按钮后做 UDP 建链”的表现，已经不再作为正式路径
