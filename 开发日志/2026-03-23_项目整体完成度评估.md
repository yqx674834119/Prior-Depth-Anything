# 2026-03-23 项目整体完成度评估

## 1. 项目目标

本项目的最终目标是：

- 使用低成本传感器组合 `ESP32 Camera + 8x8 ToF`
- 在安卓端完成 `图像 + ToF` 融合
- 生成更稠密、带尺度的深度结果
- 进一步用于 3D 场景重建与可视化

从当前仓库状态看，这个项目已经形成了比较清晰的技术主线：

1. `ESP32` 采集并发送 RGB 图像与 ToF 深度
2. `Android` 接收图像与 ToF 数据
3. `Android` 或离线脚本把 ToF 先验组织成稀疏深度 prior
4. 使用 `Depth Anything with Any Prior` 做融合推理
5. 输出深度图，并进一步生成点云结果

## 2. 两篇论文在项目中的作用

### 2.1 `2505.10565v1.pdf`

这篇论文的核心思想是：

- 单目深度预测提供 `完整但无绝对尺度` 的几何结构
- ToF 等测距传感器提供 `有尺度但稀疏/不完整` 的深度信息
- 两者融合后可以得到 `稠密、细节更好、带尺度` 的深度图

这与当前项目主线完全一致。仓库中的 [`batch_run.py`](/C:/Users/Shawn/StudioProjects/Prior-Depth-Anything/batch_run.py) 已经验证了这条路线。

### 2.2 `An_Easy_Method_for_Extrinsic_Calibration_of_Camera_and_Time-of-Flight_Sensor.pdf`

这篇论文解决的是：

- `8x8 ToF` 与 `RGB Camera` 的外参标定
- 让 ToF 的每个 zone 能够正确投影到相机图像平面

对你的项目来说，这一步的产物可以理解成：

- 一份固定的 `ToF zone -> 相机图像区域` 映射
- 在现有离线数据里，对应的就是 `fr`

也就是说，后续安卓端如果要忠实复现离线融合流程，真正缺的不是模型本身，而是这份标定映射。

## 6. 当前真正的短板

### 6.1 最大短板不是模型，而是标定

离线数据之所以能工作，是因为已经有：

- `fr`
- `hist_data`
- `mask`

其中 `fr` 本质上就是标定后的投影结果。当前安卓在线链路还没有这一步，所以在线效果和离线效果之间一定存在差距。

### 6.2 安卓端在线 prior 还是“简化版”

当前 [`DepthInferencer.kt`](/C:/Users/Shawn/StudioProjects/Prior-Depth-Anything/android_project/app/src/main/java/com/example/priordepth/logic/DepthInferencer.kt) 的做法是：

- 把 `64` 个深度值均匀放到 `8x8` 网格中心
- 直接作为 `518x518` 输入 prior

这适合作为原型验证，但还不是论文链路中的真实融合方式。

### 6.3 还没有进入“连续 3D 重建”

当前点云页本质上是：

- 单帧 RGB
- 单帧深度
- 做一个 OpenGL 点云可视化

它离“重建 3D 场景”还差至少这些环节：

- 多帧采集与缓存
- 相机/设备位姿估计
- 多帧点云配准
- 全局融合
- 网格化或可导出场景模型

## 8. 建议的下一阶段优先级

建议按下面顺序推进：

### P0：先完成标定结果固化

目标：

- 先不要追求 ESP32 端实时做标定
- 先把 `ToF zone -> 相机图像区域` 的固定映射做出来
- 形成一份可供安卓直接读取的标定文件

产出建议：

- `fr.json` 或 `fr.npy`
- 同时保留标定说明文档

### P1：安卓端按真实标定重建 prior

目标：

- 不再使用“均匀 8x8 中心点”简化摆放
- 改为根据 `fr + d + mask` 在安卓端重建稀疏先验

这一步完成后，安卓在线链路才会真正逼近离线 `batch_run.py` 的结果。

### P2：补齐在线协议

建议 ESP32 至少支持发送：

```json
{
  "d": [64个深度值],
  "mask": [64个有效位]
}
```

如果后续需要，也可以加：

- 时间戳
- 帧编号
- 传感器状态位

### P3：验证安卓端在线融合效果

目标：

- 选取固定场景
- 用离线脚本结果作为参考
- 对比安卓在线推理效果

这一步是“从能跑到可信”的关键。

### P4：再考虑真正的 3D 场景重建

当单帧融合稳定之后，再进入：

- 多帧采集
- 位姿跟踪
- 点云融合
- 网格重建

否则现在直接做 3D 重建，会把问题堆在一起。
